# DriftX

**DriftX** is an **AI reliability system concept** designed to demonstrate **drift** and **self-healing** behavior in chatbot-style interfaces.

In this project, DriftX is implemented as a **Chrome Extension (Manifest V3)** that works as a **demo layer** on top of websites like ChatGPT. It captures the latest visible AI response, simulates a drifted version of that response, and restores the original output through a healing action.

## What DriftX Does

DriftX demonstrates a simple reliability lifecycle:

1. A user asks a question to an AI chatbot.
2. The chatbot produces its normal response.
3. DriftX captures that response as the original stable state.
4. DriftX can simulate drift by replacing the visible answer with an incorrect one.
5. DriftX can heal by restoring the original captured response.

This allows DriftX to visually communicate:

- drift
- output instability
- recovery
- trust and reliability in AI systems

## Important Clarification

DriftX does **not** modify the actual ChatGPT model or backend.

The current implementation:

- does **not** retrain an AI model
- does **not** fine-tune ChatGPT
- does **not** intercept private backend logic
- does **not** perform true statistical drift detection

Instead, DriftX works at the **browser DOM level**. It changes the **visible webpage response** after it has already been generated by the chatbot.

So the best way to describe the project is:

> DriftX is the AI reliability system concept, and the Chrome extension is the demonstration interface.

## Features

- Capture the latest visible assistant response from a ChatGPT-style webpage
- Store the original response for later restoration
- Simulate drift for:
  - text responses
  - numeric responses
  - code responses
- Restore the original response using a heal action
- Display status feedback in the extension popup
- Run instantly without reloading the page
- Work without any external API dependency inside the extension

### Core Files

- `manifest.json`
  - Defines the extension configuration, permissions, popup, and content script registration.
- `popup.html`
  - Creates the popup UI shown when the extension icon is clicked.
- `popup.js`
  - Handles popup actions like opening ChatGPT, sending drift/heal commands, and updating status.
- `content.js`
  - Runs inside the webpage, captures responses, applies drift, and restores the original answer.
- `style.css`
  - Provides the popup styling and visual presentation.

## How It Works

### 1. Popup Layer

The extension popup acts as the control interface.

It provides buttons for:

- Open ChatGPT
- Drift
- Heal

### 2. Messaging Layer

When a popup button is clicked, `popup.js` sends a message to the currently active browser tab using Chrome’s tabs messaging API.

### 3. Content Script Layer

`content.js` runs on supported chatbot pages and:

- finds the latest visible assistant response in the DOM
- stores the original text and HTML
- reads the latest user prompt for context
- generates a drifted response
- restores the original response when healing is triggered

### 4. Drift Logic

DriftX uses **rule-based heuristic logic** rather than a trained model.

It detects the response type and drifts it accordingly:

- text -> wrong text answer
- numeric -> wrong numeric answer
- code -> wrong code snippet

### 5. Healing Logic

Before drift is applied, the original response is stored as a baseline.

When healing is triggered, that original stored HTML is restored, bringing back the original visible answer and formatting.

## Installation

### Load Unpacked in Chrome

1. Open Chrome and go to `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked**
4. Select this project folder
5. Pin the extension if needed

## Usage

1. Open ChatGPT in Chrome
2. Ask a question and wait for the assistant response
3. Click the DriftX extension icon
4. Use:
   - `Open ChatGPT` to open the website
   - `Drift` to simulate a wrong output
   - `Heal` to restore the original output

## Current Limitations

- Drift is **simulated**, not statistically detected in real time
- The original captured response is treated as the stable baseline
- If the original response is already wrong, healing will restore that same wrong original
- The extension currently works as a **front-end demo layer**, not a production AI middleware system
- It does not yet use:
  - KL divergence
  - PSI
  - neural networks
  - feedback loops
  - online learning
